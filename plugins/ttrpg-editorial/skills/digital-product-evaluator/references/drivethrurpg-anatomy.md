# DriveThruRPG — Anatomy of a Product Page

This reference documents the mechanical surface of DriveThruRPG that a
creator needs to understand to optimize a listing. The numbers below are
**orders of magnitude valid in early 2026**; verify current values in the
publisher dashboard or `https://help.drivethrurpg.com/` before betting
strategy on exact thresholds.

---

## 1. Product Page Anatomy

A DriveThruRPG product page has these visible surfaces, in roughly the
order a buyer encounters them:

| Surface | Purpose | Optimization lever |
|---|---|---|
| Thumbnail (search/category grid) | Capture click in scrolling browse | Cover legibility, title size |
| Hero cover (product page) | First full-size confirmation | Cover detail, genre signal |
| Title + author + price (above the fold) | Identify product instantly | Title clarity, price visibility |
| Star rating + review count | Social proof | Earn reviews early |
| Format/options selector (PDF / Softcover / Hardcover / Bundle) | Force-choice format | POD availability |
| Long description (blurb) | Convert read into buy | First-100-words discipline |
| "What's Inside" or features list | Concrete promise | Bullet structure |
| Preview button → preview PDF | Show before buy | Curated preview pages |
| Image gallery (supplementary screenshots) | Visual proof | 2–4 strong images |
| Tags / Genre / System / Publisher | Discovery + cross-listing | Tag stacking |
| Customer reviews | Social proof + objections | Reply to negative reviews |
| Customers Who Bought This Also Purchased | Cross-promotion | Bundle and pair-up |
| File details (page count, file size, type) | Technical confirmation | Page count visible elsewhere |

## 2. Category Taxonomy

DriveThruRPG uses a hierarchical category system. Each product gets one
**primary category** plus optional secondary categories. Wrong primary
category = invisible to the audience that would buy.

### Top-level TTRPG categories

- **Adventures** — playable scenarios with a beginning and end
- **Settings & Sourcebooks** — places, worlds, regions, systems
- **Genre** — Fantasy, Sci-Fi, Horror, Modern, Historical, Superhero, etc.
- **System** — D&D 5e, D&D 2024, Pathfinder, OSR systems, narrative
  systems
- **GM Tools** — generators, advice, prep tools, system-agnostic
- **Player Resources** — character sheets, options, supplements
- **Bestiaries / Monsters** — creature collections
- **Maps** — battle maps, regional maps, city maps
- **Zines** — short-form, periodicals

### Common category mistakes

- Tagging a one-shot as "Settings & Sourcebooks" because it includes
  the village it's set in → it's an **Adventure** that includes a setting,
  not the reverse. Buyers searching adventures won't find it.
- Tagging a generator as "GM Tools" when it's specifically a
  **Bestiary generator** → tag both.
- Adult content tagged only under "Adult Content" without a genre tag
  → invisible to genre browsers.

### Bestseller categories (high-volume niches as of early 2026)

- D&D 5e Adventures (1–4 level range, one-shots)
- OSR adventures (B/X, OSE, Mörk Borg, Cairn-compatible)
- Solo TTRPG (Ironsworn, Thousand Year Old Vampire-style)
- VTT-ready packs (Roll20, Foundry exports)
- Adult content (gated, smaller but loyal audience)

## 3. Tag Taxonomy

Tags are how buyers filter, and they feed DriveThruRPG's internal search.
A product should have tags from each applicable axis:

### System tags (mandatory if system-specific)

- D&D 5e (2014)
- D&D 5e (2024) — newer, sometimes both should apply
- Pathfinder 1e / 2e
- Old-School Essentials (OSE)
- B/X, BECMI
- DCC RPG
- Cairn, Mausritter, Mörk Borg (NSR / new-school-renaissance)
- Powered by the Apocalypse (PbtA)
- Forged in the Dark (FitD)
- Free League systems (Year Zero engine, etc.)
- Fate Core / Fate Accelerated
- Savage Worlds
- GURPS
- Generic / System-Neutral

### Genre tags

- Fantasy / Dark Fantasy / High Fantasy / Sword & Sorcery / Grimdark
- Horror / Cosmic Horror / Survival Horror / Gothic
- Sci-Fi / Space Opera / Cyberpunk / Post-Apocalyptic / Hard SF
- Modern / Urban / Spy / Crime / Noir
- Historical / Pulp / Weird West
- Superhero
- Mystery / Investigation / Detective
- Romance / Erotica (gated, requires Adult tag)

### Content-warning tags (use when applicable)

- Adult (NSFW; enables NSFW gating)
- Violence
- Horror Themes
- Sexual Content (adult-only)
- LGBTQ+ themes (audience signal, not warning)

### Format tags

- Adventure / Module
- One-Shot
- Campaign
- Setting
- Sourcebook
- Bestiary
- Toolkit / Generator
- Random Tables
- Zine
- Player Option (subclass, ancestry, feat, spell)
- Map Pack
- Character Sheet

### Common tag mistakes

- Selecting only the system tag and not the genre tag → product appears
  for "D&D 5e" browsers but not for "Horror" browsers
- Missing the secondary system tag for OSR-compatible products
- Not tagging the format → product surfaces for system browsers but
  not for adventure browsers
- For French products: missing the English equivalent tag → invisible
  to anglophone DTRPG market

## 4. Best-Seller Badges

DriveThruRPG awards visual badges to products based on cumulative units
sold. Badges are powerful conversion signals — buyers anchor on them.
The thresholds below are **approximate; OneBookShelf has adjusted them
over time and they vary by category**. Treat as orders of magnitude.

| Badge | Approximate units sold |
|---|---|
| Copper | ~50 |
| Silver | ~100 |
| Electrum | ~250 |
| Gold | ~500 |
| Platinum | ~1,000 |
| Mithral | ~2,500 |
| Adamantine | ~5,000+ |

**Implications for pricing strategy:**

- A product priced at $4.99 needs to sell roughly 50 units to hit Copper
  → achievable for most launches with modest promotion.
- A product priced at $24.99 needs the same 50 units → higher friction
  per unit, longer to badge, but each sale is worth more.
- PWYW with $0 minimum: downloads count toward the badge regardless of
  amount paid → fastest route to Copper / Silver, useful for visibility.

**Strategic note:** the *first* badge matters most. Going from no-badge
to Copper produces a visible conversion lift. Going from Gold to
Platinum is marginal. Front-load promotion to reach the first badge.

## 5. Preview Pages

DriveThruRPG offers two preview mechanisms:

### Default preview (auto)

The platform shows the first N pages of the product PDF. This is the
fallback, and it almost always shows:

- Cover (already seen by buyer)
- Title page
- Copyright / credits page
- Table of contents
- Forward / introduction

→ Five pages of nothing the buyer wants. Conversion-killer.

### Custom preview PDF (upload)

The creator can upload a **separate preview PDF** that replaces the
default. This is the only mechanism that controls what the buyer sees
in the preview window.

**What a strong custom preview shows (3–5 pages):**

- A characteristic full layout page (best art + clean typography)
- A scene description / read-aloud excerpt
- A stat block or mechanical element (if applicable)
- A map or handout (if applicable)
- The first page of the strongest chapter / section

**What it should NOT show:**

- Front matter (cover, copyright, ToC)
- Spoilers (final boss, plot twist resolution)
- Anything that fails the platform's NSFW threshold if the product
  itself is adult-gated

### Adult-content preview gating

For products tagged Adult / Sexual Content, the preview shown publicly
must respect DriveThruRPG's SFW threshold (no explicit imagery in the
visible preview, even if the product itself is explicit). The custom
preview PDF is the lever to comply with this without sacrificing
quality of the preview.

## 6. Image Gallery (Supplementary Screenshots)

Beyond the cover, the product page can show 1 to ~8 supplementary
images. These appear in a gallery on the listing.

**What works:**

- A full interior page screenshot (shows layout, color, density)
- A map or handout, isolated
- A stat block or mechanical excerpt
- A character portrait or location illustration
- A spread (two-page view) showing layout consistency

**What doesn't work:**

- Crops too tight to read text
- Low-resolution exports (DriveThruRPG re-compresses; submit high-res)
- Duplicate of the cover at a different angle
- AI-generated "marketing illustrations" that don't appear in the
  product itself (the buyer will feel duped)

**Optimization:** at minimum 2 supplementary images; ideal 3–4. The
gallery converts hesitant buyers who want to see what they're getting
beyond the cover.

## 7. Print on Demand (POD)

DriveThruRPG operates a POD service that bills, prints, and ships
softcover and hardcover editions on demand. Many top-selling PDF
products see total revenue **double or triple** when a POD version
is added.

### POD specs (early 2026, verify in publisher dashboard)

- **Sizes:** US Letter (8.5×11"), Digest (5.83×8.27" approximate),
  Pocket (4.25×6.87" approximate), Square (8.5×8.5", 6×6"), and a few
  variants. Available sizes depend on page count and binding.
- **Page count minimums:** softcover starts at ~32 pages, hardcover
  at ~80 pages (varies by size).
- **Binding:** Perfect bound softcover, premium hardcover (case-bound).
- **Interior:** Black & white (cheapest), Standard Color (mid), Premium
  Color (most expensive). B&W is dramatically cheaper to print → if the
  product can be designed B&W, POD margin improves significantly.
- **Cover:** Full-color glossy or matte, with optional spot UV.

### POD pricing logic

DriveThruRPG sets a **base cost** per unit (production + their margin),
and the publisher adds a markup. The buyer pays base + markup. The
publisher's revenue = markup × units sold.

- A 64-page B&W softcover digest typically has a base cost around $5–6.
- The publisher sets a retail price (e.g. $14.99) and earns the
  difference per unit.
- A 200-page Premium Color hardcover has a base cost often above $25;
  retail must clear $40+ for meaningful margin.

### POD readiness checklist

- Interior PDF: 300 DPI, embedded fonts, no bleed if no bleed setting,
  full bleed (0.125" / 3mm) if bleed setting is used.
- Cover: separate PDF with wrap-around (front + spine + back), spine
  width calculated from page count and paper weight.
- Margins: inside (gutter) at least 0.5" / 12mm for perfect bound to
  account for binding loss.
- Color profile: CMYK for color interior; B&W can be grayscale.

### When to recommend POD

- Product is **already print-ready** (designed with margins, bleed,
  color profile) → no extra work, pure revenue lift.
- Product is **30+ pages** (below this, the printed version is too
  thin to feel like a book).
- Product has **established PDF sales** showing audience demand.
- Niche supports physical (zines, settings, sourcebooks > random tables
  or single-page items).

### When to skip POD

- VTT-only product (battlemaps, tokens) — buyers don't want print.
- Product has not earned PDF sales yet — POD setup cost (time) doesn't
  return.
- Interior is designed in a way that breaks at print (full-bleed
  illustrations without bleed setting, text in margins, etc.) — fix
  the file first.

## 8. OneBookShelf Network

DriveThruRPG is part of OneBookShelf, which also operates:

- **DriveThruFiction** — fiction (novels, anthologies, short stories)
- **DriveThruComics** — comics, graphic novels
- **DriveThruCards** — card decks, accessories
- **DMs Guild** — Wizards of the Coast partner site for D&D 5e
  community content (with Wizards' Cease & Desist protection in
  exchange for revenue share)
- **Storyteller's Vault** — equivalent for World of Darkness
- **Miniature Market** (separate)

### Strategic implications

- A TTRPG product set in *Forgotten Realms* or other Wizards IP
  **must** go on DMs Guild (not DriveThruRPG) for legal coverage.
- DMs Guild has a different revenue split (50% to creator vs ~70% on
  DriveThruRPG) but legal protection for using Wizards IP.
- A fiction product (e.g. a novella set in your TTRPG world) belongs
  on DriveThruFiction, not DriveThruRPG. Cross-link in product pages.

## 9. Publisher Analytics

The publisher dashboard exposes a limited but useful dataset:

- **Sales over time** — units and revenue by day/week/month
- **Views** — how many product page views (not impressions in browse)
- **Conversion rate** — buys / views (rough estimate)
- **Customer reviews** — count, average, individual reviews
- **Top-purchased-with** — products often bought alongside (cross-sell
  intel)
- **Country breakdown** — geographic distribution of buyers

### What's missing (or limited)

- Direct funnel from thumbnail impression → click → page view → buy.
  Inference is required from views vs sales.
- No competitor benchmark data inside the dashboard.
- No keyword-level traffic data (which search terms brought buyers).

### Using analytics to diagnose post-launch failures

- **Low views relative to category peers** → discovery problem (cover,
  tags, category). Axis 3 + Axis 1 (tags).
- **Decent views, low buys** → conversion problem (blurb, price,
  preview). Axis 1 (blurb) + Axis 2 (price) + Axis 3 (preview).
- **Decent buys, no repeat customers** → catalog problem (no follow-up
  products, no bundle). Axis 4 + Axis 2 (bundles).

## 10. NSFW Gating

DriveThruRPG hosts adult content but gates it heavily.

### Gating mechanisms

- Adult products require the **Adult Content** tag.
- The buyer must enable an "Allow Adult Content" toggle in their
  account preferences to see adult products in search or browse.
- A significant portion of DriveThruRPG users have this toggle **off**
  by default → adult products are invisible to them.
- The cover and preview must respect a SFW threshold for the public
  preview surface even if the toggle is on.

### Implications for adult creators

- Audience is smaller but loyal and higher-paying per unit.
- Discovery is harder; rely on direct promotion (Patreon, social,
  newsletter) and on the "Customers Who Bought This Also Purchased"
  graph (which captures most adult-tag-toggle-on users).
- The cover must be SFW-visible at thumbnail size and on the public
  product page; an SFW alternate cover or a tasteful pose is the
  standard solution.
- The custom preview PDF (Section 5) is mandatory — the default
  preview will likely include pages that violate the SFW threshold.

## 11. Quick Sanity-Check List

Before declaring a listing "launch-ready", verify:

- [ ] Primary category set correctly
- [ ] At least one system tag (or "Generic / System-Neutral")
- [ ] At least one genre tag
- [ ] Format tag (Adventure, Setting, Zine, etc.)
- [ ] Adult Content tag if applicable
- [ ] Custom preview PDF uploaded (not default)
- [ ] 2+ supplementary screenshots in the gallery
- [ ] Cover legible at thumbnail (200×260 px)
- [ ] Title under 8 words and includes a hook
- [ ] Blurb first paragraph answers what / for whom / why
- [ ] "What's Inside" bullet list present
- [ ] Price aligns with page count benchmark
- [ ] POD considered (recommended or explicitly declined)
- [ ] At least one cross-promotion to other products in catalog (if any)
