# Output Template

Every playtest report uses this structure. The Python script generates it
mechanically; inline runs reproduce it by hand. Either way, the structure is
identical so reports across modules are comparable.

## Notes on Inline Mode

When the skill is invoked in chat without the Python script, produce the
same structure. Skip the "Generated by" footer or note "inline run, no script."

The transcript can be abbreviated in inline mode if the user wants a quick
turnaround — drop down to one paragraph per scene with inline flags, no
quoted dialogue. State that abbreviation explicitly in the verdict so the
user knows the trade-off.

## Notes on Quick Mode

Quick mode skips pass 2 (transcript) and pass 3 (persona feedback). The
report still has:

- Verdict
- Persona Coverage Map (inferred from the manuscript, not from a transcript)
- Top 5 Fixes
- What Works
- Mechanical Findings

Mark "Persona coverage inferred from manuscript, not simulation" in the map.
The coverage in quick mode is less reliable than full mode but useful for
fast iterations.

## Notes on Narrative Mode

Narrative mode skips pass 1 (mechanical) and produces:

- Verdict
- Persona Coverage Map
- Top 5 Fixes
- What Works
- Simulated Playthrough
- Per-Persona Feedback
- Cross-Pass Patterns

Useful for narrative-first systems (PbtA, FitD, generic story games) or for
modules where the mechanical layer has already been audited.
