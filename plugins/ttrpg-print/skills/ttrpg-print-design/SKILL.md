# TTRPG Print Design Skill

## Purpose
This skill provides comprehensive guidance for designing professional print-ready TTRPG documents including rulebooks, adventures, supplements, and campaign guides. It covers layout principles, typography, print specifications, visual hierarchy, and production standards for platforms like DriveThruRPG.

## When to Use This Skill
- Designing layouts for TTRPG rulebooks, adventures, or supplements
- Creating print-ready PDFs for POD (Print-on-Demand) services
- Establishing visual systems for RPG content
- Formatting stat blocks, tables, sidebars, and other RPG-specific elements
- Preparing files for professional printing or DriveThruRPG upload
- Creating style guides for TTRPG publications

## Print Specifications

### Standard Page Sizes for TTRPG Publishing

**US Letter (8.5" × 11")**
- Common for adventures and supplements
- Dimensions: 8.5" × 11" (215.9mm × 279.4mm)
- Trim size with bleed: 8.75" × 11.25"

**Digest/A5 (5.5" × 8.5")**
- Popular for indie RPG zines and compact supplements
- Dimensions: 5.5" × 8.5" (139.7mm × 215.9mm)
- Trim size with bleed: 5.75" × 8.75"

**Royal/Crown Quarto (7.44" × 9.69")**
- Common for indie RPG corebooks
- Dimensions: 7.44" × 9.69" (189mm × 246mm)
- Trim size with bleed: 7.69" × 9.94"

**A4 (8.27" × 11.69")**
- International standard, common in Europe
- Dimensions: 210mm × 297mm (8.27" × 11.69")
- Trim size with bleed: 216mm × 303mm

**6" × 9" (Trade Paperback)**
- Compact format for novels and some RPG books
- Dimensions: 6" × 9" (152.4mm × 228.6mm)
- Trim size with bleed: 6.25" × 9.25"

**8.5" × 11" Landscape**
- Used for battlemaps, DM screens, or special products
- Dimensions: 11" × 8.5" (279.4mm × 215.9mm)
- Trim size with bleed: 11.25" × 8.75"

### Bleed, Margins, and Safe Zones

**Bleed**
- Standard bleed: 0.125" (3mm) on all sides
- DriveThruRPG requirement: 0.125" minimum
- Extends background colors/images beyond trim line
- Prevents white edges after cutting

**Margins (Inside Safe Zone)**
- Outer margin: 0.5" - 0.75" (12-19mm)
- Inner margin (gutter): 0.75" - 1" (19-25mm) for perfect bound
- Inner margin: 0.5" - 0.625" for saddle-stitched
- Top margin: 0.5" - 0.75" (12-19mm)
- Bottom margin: 0.625" - 0.875" (16-22mm)
- Add extra 0.125" - 0.25" to gutter for books over 200 pages

**Safe Zone (Critical Content Area)**
- Keep all important text and elements at least 0.25" (6mm) inside trim line
- Critical text (page numbers, titles): 0.375" - 0.5" from trim
- Headers and footers: 0.5" from top/bottom trim

### Resolution and Color

**Resolution**
- Text and vector graphics: 300 DPI minimum
- Raster images (photos, illustrations): 300 DPI at final size
- Large background textures: 150-300 DPI acceptable
- Never use images below 150 DPI

**Color Mode**
- CMYK for print (not RGB)
- Rich black: C:60, M:40, Y:40, K:100 (not pure K:100)
- Avoid pure black backgrounds (causes show-through)
- Total ink coverage: Keep under 300% for coated stock, 280% for uncoated

**PDF Export Settings**
- PDF/X-1a:2001 or PDF/X-4 standard
- Embed all fonts
- Compress images: JPEG quality 80-100
- No security settings or passwords
- Flatten transparency
- Single-page PDFs for DriveThruRPG POD

## Typography for TTRPG Documents

### Font Selection Principles

**Body Text Fonts**
- Use serif fonts for extended reading (rulebooks, adventures)
- Recommended: Garamond, Caslon, Crimson Text, EB Garamond, Libre Baskerville
- Sans-serif acceptable for modern or sci-fi settings
- Font size: 9-11pt for body text
- Leading (line spacing): 120-145% of font size
- Character spacing: -5 to +5 tracking for optimal readability

**Heading Fonts**
- Display or decorative fonts for flavor
- Ensure readability at various sizes
- Can mix serif body with sans-serif headings or vice versa
- Create clear visual hierarchy through size, weight, and spacing

**Special Use Fonts**
- Stat blocks: Often sans-serif or condensed serif for space efficiency
- Tables: Condensed or sans-serif for density
- Flavor text: Italic or script fonts (use sparingly)
- Handwriting fonts: Use only for genuine handout reproduction

### Type Hierarchy

**Six-Level Hierarchy for RPG Books**

1. **Chapter Titles (H1)**
   - Size: 24-36pt
   - Weight: Bold or Heavy
   - Spacing: Extra tracking (50-200)
   - Position: Often with decorative elements

2. **Major Section Headings (H2)**
   - Size: 18-24pt
   - Weight: Bold or Semibold
   - Spacing: 1.5-2× line spacing above, 0.5-1× below

3. **Subsection Headings (H3)**
   - Size: 14-16pt
   - Weight: Bold or Semibold
   - Spacing: 1.5× line spacing above, 0.5× below

4. **Minor Headings (H4)**
   - Size: 11-13pt
   - Weight: Bold or All Caps
   - Spacing: 1× line spacing above and below

5. **Inline Headings (H5)**
   - Size: Same as body text
   - Weight: Bold or Italic
   - Runs into paragraph

6. **Body Text**
   - Size: 9-11pt
   - Weight: Regular
   - Leading: 120-145%

### Specialized Typography

**Stat Blocks**
- Header: Bold, 10-12pt, often with background color
- Labels: Bold or small caps, 8-9pt
- Values: Regular, 8-9pt
- Tight leading: 100-110% for space efficiency
- Use rules/lines to separate sections
- Background fill: 5-10% tint for distinction

**Tables**
- Header row: Bold, 9-10pt, often with background
- Body cells: Regular, 8-9pt
- Alternating row tints: 5% difference for readability
- Adequate cell padding: 3-6pt on all sides
- Vertical rules: Use sparingly or not at all
- Horizontal rules: Between header and body, optionally between rows

**Sidebars and Callouts**
- Background tint: 5-15% of accent color
- Border: 0.5-2pt, optional decorative corners
- Internal margins: 0.25-0.5" on all sides
- Font size: Same as body or 1pt smaller
- Italic or different font for distinction

## Layout and Visual Design

### Grid Systems

**Column Grids**
- Single column: 4-6" width optimal for readability (45-75 characters)
- Two columns: Common for rulebooks and dense content
- Three columns: For reference material, tables, or magazine-style
- Gutter width: 0.25-0.5" between columns

**Baseline Grid**
- Set leading increment (usually 12pt or 14pt)
- Align all text baselines to grid
- Images and rules align to baseline grid
- Maintains vertical rhythm across spreads

**Modular Grid**
- Divides page into modules (e.g., 6×8 grid)
- Allows flexible placement of images and text blocks
- Good for varied content types (adventure modules)

### Page Layout Principles

**Facing Pages (Spreads)**
- Design spreads as units, not individual pages
- Balance visual weight across the spread
- Keep related content on the same spread when possible
- Mirror margins: Inner margins should align

**Visual Flow**
- Eye naturally enters top-left, exits bottom-right
- Place important content in natural flow path
- Use visual elements to guide eye movement
- Break up text walls with images, sidebars, or whitespace

**Whitespace (Negative Space)**
- Essential for readability and elegance
- Margins are active design elements, not wasted space
- Use whitespace to group related content
- Avoid cramming; let design breathe

**The Rule of Thirds**
- Divide page into 3×3 grid
- Place focal points at intersections
- Creates dynamic, interesting layouts
- Especially useful for image placement

### Images and Illustrations

**Image Placement**
- Bleed images to edge or use clear borders
- Avoid trapping images between text blocks awkwardly
- Wrap text around images with adequate spacing (0.125-0.25")
- Group caption with image (never separated across pages)

**Image Sizing**
- Full-page art: Edge-to-edge bleed
- Half-page: 50% of text height with margins
- Quarter-page: 25% of text height
- Spot illustrations: 1-3" square, aligned to baseline grid

**Art Styles for TTRPG**
- Consistent style throughout publication
- Mix of full-page showpiece art and spot illustrations
- Border treatments can add flavor (ornate frames, torn edges, etc.)
- Silhouetted art useful for flexible placement

### Color and Theming

**Color Palette Strategy**
- Primary color: Main theme color (chapter headers, accent elements)
- Secondary color: Supporting color for variation
- Neutral base: Black/dark gray text, cream/white background
- Tint scales: 10%, 25%, 50%, 75%, 100% of primary for backgrounds

**Color Psychology for RPG Genres**
- Fantasy: Deep purples, rich reds, forest greens, gold accents
- Horror: Deep reds, blacks, grays, sickly greens
- Sci-fi: Blues, cyans, silvers, stark blacks and whites
- Modern/Urban: Grays, blues, industrial colors
- Historical: Sepia tones, aged paper colors, muted earth tones

**Accessibility Considerations**
- Text contrast: Minimum 4.5:1 ratio (7:1 for AAA)
- Avoid red-green combinations (colorblind accessibility)
- Don't rely on color alone for information
- Test in grayscale to ensure readability

### Decorative Elements

**Rules and Borders**
- Page borders: 0.5-2pt, ornate or simple based on theme
- Section dividers: 0.5-1pt rules with decorative ends
- Text boxes: 1-3pt borders with rounded or decorative corners
- Consistency: Use same style throughout document

**Ornamental Elements**
- Corner flourishes for chapter starts or special sections
- End-of-section glyphs (symbols to mark breaks)
- Decorative initial caps (drop caps) for chapter starts
- Watermarks: 5-10% opacity, non-intrusive

**Background Textures**
- Subtle paper textures: 5-15% opacity
- Parchment or aged paper for fantasy settings
- Avoid overly busy backgrounds that reduce readability
- Keep backgrounds light enough for easy reading

## TTRPG-Specific Design Elements

### Stat Blocks

**Classic D&D Style**
- Beige/tan background (10-20% fill)
- Bold header with creature name and size/type/alignment
- Horizontal rules separating sections
- Two-column layout for attributes
- Ability scores in bold with modifiers
- Actions section with hanging indents
- Compact, information-dense design

**Modern Minimalist Style**
- Clean white or light gray background
- Thin borders (0.5-1pt)
- Ample whitespace between sections
- Clear typography hierarchy
- Icons for quick reference (optional)

**Narrative Style**
- Integrated into prose
- Key stats highlighted in bold or different color
- Less mechanical, more story-focused
- Useful for NPCs in narrative adventures

### Sidebars and Callouts

**Types of Sidebars**
- **Lore boxes**: Background information, world flavor
- **Tactical notes**: Combat tips, strategic advice
- **DM notes**: Behind-the-scenes guidance
- **Optional rules**: Variant mechanics
- **Read-aloud text**: Boxed text for DMs

**Design Treatment**
- Background color: 5-15% tint
- Border: 1-2pt, optional corner ornaments
- Icon or header graphic to indicate type
- Internal padding: 0.25-0.5" margins
- Font: Italic, different font, or standard with colored header

### Tables

**Combat Tables**
- Clear headers with background color
- Alternating row colors for easy scanning
- Adequate column width for content
- Bold column headers
- Minimal vertical rules

**Random Tables**
- Die roll column (d6, d20, etc.) in bold
- Result column with adequate width
- Optional: Color-code by probability ranges
- Consider two-column layout for long tables

**Reference Tables**
- Comprehensive borders for complex data
- Color-coded headers for different stat types
- Footnotes below table for exceptions
- Page number in header if multi-page

### Chapter and Section Starts

**Chapter Opener Design**
- Full-page or half-page spread
- Large chapter number (decorative)
- Chapter title in display font
- Optional: Quote, illustration, or summary
- Plenty of whitespace

**Section Breaks**
- Consistent heading style
- Optional: Small decorative element
- Extra space above (2-3× normal)
- May include small illustration or icon

## Production and Technical Considerations

### Document Setup

**Page Setup Checklist**
- [ ] Correct page size with bleed
- [ ] Facing pages enabled for spreads
- [ ] Margins set with gutter consideration
- [ ] Baseline grid established
- [ ] Master pages created for consistent elements
- [ ] Color mode set to CMYK
- [ ] Resolution set to 300 DPI for all images

**Master Pages**
- Create masters for: front matter, body pages, chapter openers, back matter
- Include: headers/footers, page numbers, background elements
- Override masters locally only when necessary
- Maintain consistency across document

### File Organization

**Layer Management**
- Background layer: Textures, watermarks, page borders
- Content layer: Text, images, main elements
- Guides layer: Margins, grids, alignment guides
- Notes layer: Production notes, bleeds marks (turn off for export)

**Asset Management**
- Link external images (don't embed) during design
- Organize assets in clear folder structure
- Use naming conventions: "chapter03_monster_goblin_01.png"
- Keep original high-res files separate from working files

### Pre-Flight Checklist

**Before Exporting**
- [ ] All fonts embedded or outlined
- [ ] All images 300 DPI and CMYK
- [ ] No spot colors unless intentional
- [ ] Rich black used instead of pure black
- [ ] All text in safe zone
- [ ] Bleed elements extend to bleed line
- [ ] No RGB elements
- [ ] Links to images intact
- [ ] Spell check complete
- [ ] Page numbers sequential and correct

**PDF Export**
- PDF/X-1a:2001 or PDF/X-4
- Compression: JPEG, High quality (80-100)
- Embed all fonts
- Include bleed settings (0.125")
- Flatten transparency
- No security settings
- Single pages (not spreads) for POD

### DriveThruRPG Specific Requirements

**POD Requirements**
- PDF/X-1a:2001 or PDF/X-4 format
- 0.125" bleed on all sides
- CMYK color mode only
- 300 DPI minimum for images
- Embed all fonts
- Single-page PDFs (not spreads)
- Total page count must be divisible by 2 for saddle-stitch
- Cover as separate file (template available from DTRPG)

**Cover Design**
- Use DTRPG cover template for exact specifications
- Front cover, spine, back cover in one file
- Spine width calculator available on DTRPG
- Barcode placement area on back cover (if using ISBN)
- 0.125" bleed on all sides
- Safe zone: 0.125" from trim line

## Common TTRPG Design Patterns

### Chapter Organization

**Standard Structure**
1. Chapter opener (full or half page)
2. Introduction paragraph
3. Major sections with H2 headings
4. Subsections with H3-H4 headings
5. Sidebars and callouts as needed
6. Tables and stat blocks integrated into flow
7. Chapter summary or conclusion

**Visual Rhythm**
- Alternate text-heavy and image-heavy pages
- Break up long sections with sidebars or images
- Use pull quotes to add visual interest
- Maintain consistent spacing between elements

### Page Elements

**Headers and Footers**
- Running header: Chapter or section name
- Page numbers: Outer corners or centered bottom
- Small decorative element (optional)
- Different style for chapter openers

**Page Numbers**
- Front matter: Lowercase Roman numerals (i, ii, iii)
- Body: Arabic numerals (1, 2, 3)
- Position: Outer corners or centered bottom
- Size: 8-10pt, often in different font than body

### Two-Page Spread Design

**Left Page (Even)**
- Natural entry point on inner edge
- Consider placing important info here
- Headers on left side
- Images can bleed to left edge

**Right Page (Odd)**
- Natural exit point on outer edge
- Strong visual elements here catch attention
- Headers on right side
- Images can bleed to right edge

**Cross-Spread Elements**
- Maps or large images can span both pages
- Leave 0.25-0.5" gutter gap for binding
- Mirror image slightly to account for binding curve
- Ensure no critical details in gutter

## Software-Specific Guidance

### Affinity Publisher
- Set up baseline grid: View > Show Baseline Grid
- Master pages: Pages panel > New Master
- Bleed setup: File > Document Setup > Bleed
- PDF export: File > Export > PDF/X-1a

### Adobe InDesign
- Baseline grid: Preferences > Grids
- Master pages: Pages panel, A-Master
- Bleed: Document Setup (⌘⌥P)
- Preflight: Window > Output > Preflight
- PDF export: File > Export > PDF (Print)

### Scribus (Open Source)
- Bleed: File > Document Setup > Bleeds
- Master pages: Edit > Master Pages
- PDF export: File > Export > Save as PDF > PDF/X-3

## Design Philosophy

### Principles of Good TTRPG Design

1. **Usability First**: Design for the table, not just the eye
   - Easy to scan during play
   - Clear information hierarchy
   - Quick reference elements accessible

2. **Consistency**: Establish patterns and stick to them
   - Same style for same element types
   - Predictable placement
   - Unified visual language

3. **Clarity Over Cleverness**: Don't sacrifice readability for style
   - Decorative elements support, don't distract
   - Information easy to find and parse
   - White space enhances, not wastes

4. **Appropriate Theming**: Match design to content
   - Fantasy: Ornate, traditional, organic
   - Horror: Stark, unsettling, dramatic contrast
   - Sci-fi: Clean, geometric, modern
   - Let theme enhance immersion

5. **Professionalism**: Quality execution signals quality content
   - Proper typography
   - Correct print specifications
   - Attention to detail
   - Polish in every element

## Testing and Iteration

### Design Review Process

**Self-Review Questions**
- Is the hierarchy clear at a glance?
- Can I find information quickly?
- Does it work in black and white?
- Are spreads balanced?
- Is the design consistent throughout?
- Does it match the genre and tone?

**Print Testing**
- Print sample pages on home printer
- Check readability at actual size
- Verify colors (though they won't match exactly)
- Test with page protectors (simulates glossy coating)
- Check bleed by trimming printed samples

**User Testing**
- Have someone unfamiliar navigate the document
- Time how long it takes to find specific information
- Ask about visual appeal and readability
- Get feedback from actual play use

## Output and Delivery

When providing design recommendations using this skill, Claude should:

1. **Assess the project scope**: Page count, content type, target audience, genre
2. **Recommend appropriate specifications**: Page size, margins, resolution
3. **Suggest typography**: Font pairings, sizes, hierarchy
4. **Provide layout guidance**: Grid system, visual flow, element placement
5. **Offer style direction**: Color palette, decorative elements, theming
6. **Technical checklist**: Pre-flight items, export settings, DTRPG requirements
7. **Give practical examples**: Specific measurements, CMYK values, tool settings

Format recommendations in clear, actionable prose that balances professional standards with practical creativity. Include specific measurements and technical values rather than vague descriptions.
